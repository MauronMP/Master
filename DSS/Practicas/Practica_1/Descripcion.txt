Importante: el espacio de nombres de la API cambió de javax.* a jakarta.*. Esto puede manifestarse de muchas maneras:

-El software escrito para Servlet 4.0 no compila contra los jars de API de Tomcat 10; es decir, el Servlet 5.0 JAR lanza error de compilación en javax.servlet.* pero Servlet 4.0 JAR no lo hace,

-Las aplicaciones Servlet 4.0 que utilizan un descriptor 'web.xml' lanzarán muchas excepciones: ClassNotFoundExceptions y no arrancan; La ejecución con Tomcat 10.x producirá java.lang.NoClassDefFoundError en javax/servlet/ServletRequestListener.

-Las aplicaciones Servlet 4.0 que utilizan un descriptor web.xml registran un error "X is not a jakarta.servlet.Servlet", si se programa utilizando los servicios del servlet y producirá el error:  Servlet class org.restlet.ext.servlet.ServerServlet is not a jakarta.servlet.Servlet.

-Todas las aplicaciones Servlet 4.0 que utilizan anotaciones para declarar servlets dejarán de funcionar.

-Las aplicaciones Servlet 4.0 que dependen de un ServletContainerInitializer (como las aplicaciones Spring y Spring Boot) no arrancarán. Por ejemplo, desplegar Spring MVC 5 en Tomcat 10  indicará un error: "deployment problems".
